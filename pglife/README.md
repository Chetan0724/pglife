# PG Life

PG Life is a modern web application that helps users find and book PG (Paying Guest) accommodations. The platform connects property owners with potential tenants, making the process of finding and managing PG accommodations seamless and efficient.

## Features

- **User Authentication**: Secure signup and login functionality
- **Property Listings**: Browse through available PG accommodations with detailed information
- **Advanced Search**: Filter properties by location, price, amenities, and more
- **Property Management**: Property owners can list and manage their properties
- **Subscription Plans**: Access premium features with subscription plans
- **Admin Dashboard**: Comprehensive dashboard for managing users, properties, and payments
- **Multi-language Support**: Support for multiple languages
- **Responsive Design**: Works seamlessly across all devices

## Tech Stack

- **Frontend**: React with TypeScript
- **Styling**: Tailwind CSS with shadcn/ui components
- **State Management**: React Context API
- **Backend**: Supabase (Authentication, Database, Storage)
- **Payment Integration**: Razorpay
- **Deployment**: Vercel

## Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm or yarn
- Supabase account
- Razorpay account (for payment integration)

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/pglife.git
   cd pglife
   ```

2. Install dependencies:
   ```bash
   npm install
   # or
   yarn install
   ```

3. Create a `.env` file in the root directory and add your environment variables:
   ```env
   VITE_SUPABASE_URL=your_supabase_url
   VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
   VITE_RAZORPAY_KEY_ID=your_razorpay_key_id
   ```

4. Start the development server:
   ```bash
   npm run dev
   # or
   yarn dev
   ```

5. Open [http://localhost:5173](http://localhost:5173) in your browser to see the application.

### Database Setup

1. Create the following tables in your Supabase database:
   - users
   - properties
   - subscriptions
   - payments
   - property_images

2. Set up the necessary policies for row-level security.
